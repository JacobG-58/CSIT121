Labs 1 & 2

The process of learning what HTML is and how to apply what I've learned to create my own webpages has been an enjoyable experience. After going through the learning material and doing a bit of research, I had a decent idea of how to approach utilizing HTML to build a webpage. While my knowledge of the HTML syntax is still on a basic level of comprehension, I think my limited experience with programming languages has helped in the memorization aspect of basic tags and structures. If I was having trouble remembering information, I would reference the Element Reference Website that was provided in the learning material.

What I learned

  1. Tags are declared by the bracket symbols surrounding them.
  2. The format for a basic HTML page includes:
    (Doctype Declaration, Head, Title, Body, Headers, Paragraphs, and Closures Of All Tags Listed.)
  3. To close a tag, you must add a forward slash (/) before its signifying letters.
    Example: "head" closes with "/head"

Lab 3

I have spent some time learning about the basics of CSS and how it can be effectively used to alter the styling of a web page, thus giving the user a more enjoyable experience. Similarly to HTML, the syntax is composed of elements that are used to assign various styling options to different assets. CSS elements have been easier to remember for me because their names are more descriptive and the parameters within each element are fairly straightforward. However, it can become tricky when determining what elements should be assigned to which text, and can get overwhelming if you want a higher level of stylish diversity. This is where selectors come in; id and class selectors make a world of difference when trying to diversify style in your webpage because they allow you to set styling rules that can be applied specifically to certain parts of your HTML file.

What I learned

  1. Class selectors must come after a "." and id selectors must come after a "#".
  2. A style sheet is a file referenced directly by an HTML file, containing rules and parameters that define the styling choices for a web page.
  3. When creating borders, it is often a good idea to follow them up with padding style changes to ensure the content within the borders is not cramped and messy looking.

Citations:

  • Learn About Selectors
  • CSS Element Reference Sheet

Lab 5

In this lab, I focused on responsive design using Flexbox and media querie. focused on how to create layouts that adapt to different screen sizes (desktop/ phone) while keeping everything easy to read. This lab also put strong emphasis on accessibility by ensuring a clean layout, consistent typography, and balanced color choices make a page easier to read and navigate. To achieve this level of accessibility it is also important to have a functional navigation pane. This allows users to efficiently navigate to parts of the website they may not originally see if on certain devices.

What I learned

  1. Responsive design ensures websites look good on all devices.
  2. Media queries can be used to change layouts based on screen width.
  3. Navigation anchors improve website accesibility by allowing quick access to page sections.
